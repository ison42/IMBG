# MBTI + 五行性格分析网站设计文档

## 1. 项目概述
这是一个手机端的MBTI + 五行性格分析网站，采用趣味游戏风格，用户输入MBTI类型和出生日期后，通过一道校准题目，生成个性化的性格分析结果。

## 2. 核心功能

### 2.1 功能模块
1. **角色创建**：用户选择MBTI类型和出生日期
2. **冒险测试**：随机生成一道校准题目
3. **英雄档案**：生成性格分析结果
4. **数据存储**：将测试结果记录到数据库

### 2.2 技术架构
- **前端**：React + TypeScript + Tailwind CSS
- **后端**：Supabase (Auth, Database, Storage)
- **数据库**：Supabase PostgreSQL

## 3. 数据库设计

### 3.1 表结构

#### `test_results` 表
| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| `id` | `UUID` | `PRIMARY KEY` | 测试结果ID |
| `mbti_type` | `VARCHAR(4)` | `NOT NULL` | MBTI类型 |
| `birth_date` | `DATE` | `NOT NULL` | 出生日期 |
| `wuxing_type` | `VARCHAR(10)` | `NOT NULL` | 五行类型 |
| `calibration_answer` | `VARCHAR(1)` | `NOT NULL` | 校准题目答案 |
| `result_summary` | `TEXT` | `NOT NULL` | 结果摘要 |
| `created_at` | `TIMESTAMP` | `DEFAULT NOW()` | 创建时间 |
| `ip_address` | `VARCHAR(50)` | | 用户IP地址（用于统计） |
| `user_agent` | `TEXT` | | 用户代理（用于统计） |

### 3.2 数据安全
- 不存储用户个人身份信息
- 仅存储测试相关数据
- 对IP地址进行匿名化处理
- 遵循GDPR和相关隐私法规

## 4. 前端实现

### 4.1 页面流程
1. **角色创建页**：选择MBTI类型和出生日期
2. **冒险测试页**：回答随机校准题目
3. **英雄档案页**：展示性格分析结果

### 4.2 数据存储流程
1. 用户完成测试后，前端收集测试数据
2. 通过Supabase SDK将数据发送到数据库
3. 数据库记录测试结果
4. 前端显示成功提示

### 6.1 前端
- 使用React hooks管理状态

### 6.2 后端
- Supabase数据库配置

### 6.3 五行计算
